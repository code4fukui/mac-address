# mac-address

このリポジトリには、観光客の行動分析を目的として、金沢大学がWi-Fiパケットセンシングにより取得したオープンデータが含まれています。

本リポジトリは、北陸インバウンド観光DX・データコンソーシアムによって公開されています。

## データについて

生データは、さまざまな観光施設でキャプチャされたWi-Fiプローブリクエストで構成されています。各レコードはデバイスの検出イベントを表します。

-   **ユーザー識別子 (`ユーザー情報`)**: ユーザーのMACアドレスはSHA-256ハッシュを用いて匿名化されています。
-   **データ構造**: 生データは、施設および日付ごとに以下の構造で整理されています。
    ```
    ./[FacilityID]_[FacilityName]/[YYYYMMDD]/[YYYYMMDD].csv
    ```
    例: `27_Higashichaya_Gold_Leaf_Sakuda/20241010/20241010.csv`

### 生データのカラム
-   `時間`: 検出イベントのタイムスタンプ。
-   `施設No`: イベントが記録された施設の一意のID。
-   `ユーザー情報`: 検出されたデバイスのハッシュ化されたMACアドレス。

## 加工済みデータセット

提供されているスクリプトを使用することで、生データから以下の集計済みデータセットを生成できます。

-   [user.csv](user.csv): 各ユニークユーザーの概要。施設間の移動ルートが含まれます。
-   [move.csv](move.csv): 全ユーザーの移動を統合したログ。同一ユーザーが同一施設で30分以内に検出された場合、それらは単一のエントリに集約され、滞在時間が計算されます。
-   [move/](move): `move.csv`と同じ移動データですが、施設ごとに個別のCSVファイルに分割されています。

### データ概要
-   **生データ総件数**: 30,794,029
-   **ユニークユーザー総数**: 3,651

## 加工済みデータセットの生成

処理スクリプトを実行するには[Deno](https://deno.land/)が必要です。以下のコマンドを順番に実行してください。

1.  **生データを処理しユーザー別のファイルを作成:**
    このスクリプトは生データを読み込み、30分以内のレコードを単一の滞在イベントに統合して、`user/`ディレクトリに中間ファイルを作成します。
    ```sh
    deno run -A make.js
    ```

2.  **ユーザーと移動データセットを生成:**
    このスクリプトは中間ファイルを集約して`user.csv`と`move.csv`を生成します。
    ```sh
    deno run -A make2.js
    ```

3.  **施設レベルの移動データを生成:**
    このスクリプトは`move.csv`を施設ごとのファイルに分割し、`move/`ディレクトリに配置します。
    ```sh
    deno run -A separatemove.js
    ```
